# 会议数据集使用说明

本目录目前包含两类会议数据集：

1. **AliMeeting Eval**：中文会议音频 + 转写标注。
2. **AliMeeting4MUG**：中文会议理解与摘要任务数据。

数据目录：

```text
D:\HACKERMarathon\Project\dataset
```

---

## 1. 数据集文件结构

### 1.1 AliMeeting Eval

文件：

```text
D:\HACKERMarathon\Project\dataset\Eval_Ali.tar.gz
```

用途：

- 中文会议语音识别 ASR
- 说话人分离 / speaker diarization
- 远场会议语音处理
- 会议音频转写评测

解压：

```powershell
cd D:\HACKERMarathon\Project\dataset
tar -xzf Eval_Ali.tar.gz
```

解压后结构大致为：

```text
Eval_Ali/
  Eval_Ali_far/
    audio_dir/      # 远场 8 通道会议音频 .wav
    textgrid_dir/   # 远场对应转写、时间戳 .TextGrid
  Eval_Ali_near/
    audio_dir/      # 近场耳麦音频 .wav
    textgrid_dir/   # 近场对应转写、时间戳 .TextGrid
```

说明：

- `far`：远场麦克风阵列音频，更接近真实会议室场景。
- `near`：近场耳麦音频，语音质量通常更好。
- `.wav`：音频文件。
- `.TextGrid`：Praat 标注格式，包含说话时间段和转写内容。

## 2. AliMeeting4MUG 数据格式

每个 CSV 通常是两列：

```text
idx	content
```

其中：

- `idx`：样本编号。
- `content`：JSON 字符串，保存一场会议的结构化信息。

`content` 解析后常见字段：

| 字段 | 含义 |
|---|---|
| `meeting_key` | 会议 ID |
| `sentences` | 会议逐句转写，包含说话人、时间戳和句子文本 |
| `candidate` | 会议级摘要句、关键词候选 |
| `topic_segment_ids` | 话题分割点及话题级标题/摘要候选 |
| `paragraph_segment_ids` | 段落分割句子 ID |
| `action_ids` | 行动项句子 ID |

`sentences` 中每句话一般包含：

| 字段 | 含义 |
|---|---|
| `id` | 句子 ID |
| `speaker` | 说话人编号 |
| `start_time` | 开始时间，单位秒 |
| `end_time` | 结束时间，单位秒 |
| `s` | 句子文本 |

## 3. 读取 AliMeeting4MUG

### 3.1 读取一个样本

```python
import csv
import json
from pathlib import Path

csv_path = Path(r"D:\HACKERMarathon\Project\dataset\AliMeeting4MUG\extracted\train\train.csv")

with csv_path.open("r", encoding="utf-8", newline="") as f:
    reader = csv.DictReader(f, delimiter="\t")
    row = next(reader)

item = json.loads(row["content"])

print(item["meeting_key"])
print(item.keys())
print(item["sentences"][0])
```

---

### 3.2 拼接整场会议文本

```python
transcript = "\n".join(
    f'{s["speaker"]}: {s["s"]}'
    for s in item["sentences"]
)

print(transcript[:1000])
```

输出形式示例：

```text
no.2: 大家先看一下这个问题
no.3: 好的
no.1: 我补充一点
```

---

### 3.3 提取会议级抽取式摘要

AliMeeting4MUG 的摘要是**抽取式摘要**：摘要由原会议中的若干句子组成。

```python
sent_by_id = {str(s["id"]): s for s in item["sentences"]}

summary_ids = item["candidate"][0]["key_sentence"]

summary = "\n".join(
    sent_by_id[i]["s"]
    for i in summary_ids
    if i in sent_by_id
)

print(summary)
```

说明：

- `candidate[0]["key_sentence"]` 是一组被选为摘要的句子 ID。
- 可把这些句子作为摘要标签。

---

### 3.4 提取关键词

```python
keywords = item["candidate"][0].get("key_word", [])
print(keywords)
```

---

### 3.5 提取话题分割点

```python
topic_boundaries = [x["id"] for x in item.get("topic_segment_ids", [])]
print(topic_boundaries)
```

这些 `id` 通常表示话题边界句。

---

### 3.6 提取话题级标题和摘要句

```python
for topic in item.get("topic_segment_ids", []):
    boundary_id = topic["id"]
    print("Topic boundary:", boundary_id)

    for cand in topic.get("candidate", []):
        print("Title:", cand.get("title"))
        print("Key sentences:", cand.get("key_sentence"))
```

---

### 3.7 提取行动项

```python
sent_by_id = {str(s["id"]): s for s in item["sentences"]}
action_ids = [str(x["id"]) for x in item.get("action_ids", [])]

actions = [
    sent_by_id[i]["s"]
    for i in action_ids
    if i in sent_by_id
]

print(actions)
```

## 5. 典型任务构造方式

### 5.1 会议摘要任务

输入：

```text
sentences 拼接成的会议全文
```

标签：

```text
candidate[0]["key_sentence"] 对应的原句文本
```

示例：

```python
source_text = "\n".join(s["s"] for s in item["sentences"])

sent_by_id = {str(s["id"]): s for s in item["sentences"]}
target_summary = "\n".join(
    sent_by_id[i]["s"]
    for i in item["candidate"][0]["key_sentence"]
    if i in sent_by_id
)
```

---

### 5.2 话题分割任务

输入：

```text
sentences 列表
```

标签：

```text
topic_segment_ids 中的 id
```

示例：

```python
boundary_ids = {x["id"] for x in item.get("topic_segment_ids", [])}

samples = []
for s in item["sentences"]:
    samples.append({
        "sentence": s["s"],
        "is_topic_boundary": s["id"] in boundary_ids,
    })
```

---

### 5.3 关键词抽取任务

输入：

```text
会议全文
```

标签：

```text
candidate[0]["key_word"]
```

示例：

```python
source_text = "\n".join(s["s"] for s in item["sentences"])
target_keywords = item["candidate"][0].get("key_word", [])
```

---

### 5.4 行动项检测任务

输入：

```text
单句文本
```

标签：

```text
该句 id 是否在 action_ids 中
```

示例：

```python
action_ids = {x["id"] for x in item.get("action_ids", [])}

samples = []
for s in item["sentences"]:
    samples.append({
        "sentence": s["s"],
        "speaker": s["speaker"],
        "start_time": s["start_time"],
        "end_time": s["end_time"],
        "is_action_item": s["id"] in action_ids,
    })
```

## 6. 推荐使用顺序

### 如果目标是做会议摘要

优先使用：

```text
AliMeeting4MUG\extracted\train\train.csv
AliMeeting4MUG\extracted\dev\dev.csv
```

训练：

- 输入：会议全文。
- 输出：抽取式摘要句。

---

### 如果目标是做语音识别或说话人分离

优先使用：

```text
Eval_Ali\Eval_Ali_far\audio_dir
Eval_Ali\Eval_Ali_far\textgrid_dir
```

训练或评测：

- 输入：`.wav`。
- 标签：`.TextGrid` 中的时间戳和文本。

---

### 如果目标是音频到会议摘要

需要两步：

1. 用 AliMeeting Eval 做 ASR / 转写。
2. 用 AliMeeting4MUG 做会议文本摘要训练。

注意：

- AliMeeting Eval 和 AliMeeting4MUG 不一定是同一批会议的一一对应数据。
- 更实际的做法是先训练/调用 ASR 得到会议文本，再把会议文本送入摘要模型。

---

## 7. 注意事项

1. `*_without_label.csv` 是无标签测试集，只适合推理，不适合训练。
2. 优先用 `train.csv` 训练，用 `dev.csv` 验证。
3. AliMeeting4MUG 的摘要是抽取式摘要，不是人工自由生成摘要。
4. AliMeeting Eval 音频包较大，解压后会占用更多磁盘空间。
5. 如果终端显示中文乱码，优先确认 Python 以 `utf-8` 读取；部分字段若仍乱码，可能是源数据本身编码异常。
6. ModelScope token 已出现在终端历史和对话中，用完后建议去 ModelScope 个人中心重置。
